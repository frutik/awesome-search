# Affordances for Conversational Search

## The Problem

In application design, **affordances** are visual/interactive cues guiding users in discovering how to interact. Traditional search interfaces use recognizable affordances (buttons, links, filters).

Conversational interfaces lack these cues. As design critic Amelia Wattenberger noted: "Text inputs have no affordances."

Users face uncertainty about:
- How to begin expressing their information needs
- What content exists and is available
- How to effectively formulate queries

Conversational search also abandons **autocomplete** and **faceted search/filters** — features modern users expect.

## Proposed Solutions

**Initial Guidance Through Examples**
Display example queries to orient users. ChatGPT provides static examples; Bard randomizes from a curated set for diversity.

**Suggestions as Natural Language Refinement**
Offer contextual follow-up suggestions instead of structured facets. Example:
- Query: "What is a good gift for a cat lover?"
- Suggestions: "What are good cat toys?", "What's a good gift under $50?"

This mirrors faceted search within natural language. Shopify's shopping assistant exemplifies this by asking clarifying questions about occasion and budget.

## Current Status

Conversational search remains developmental. The challenge: balancing natural language flexibility against user burden — providing sufficient guidance without constraining expressive freedom.
