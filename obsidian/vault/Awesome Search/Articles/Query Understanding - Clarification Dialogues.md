# Clarification Dialogues

Part of the **Query Understanding** series by [[Daniel Tunkelang]].

## Overview

When a query is genuinely ambiguous and the cost of guessing wrong is high, the system can ask the user a targeted question rather than hedging its results. Clarification dialogues turn what would otherwise be a one-shot query into a brief exchange that progressively narrows the search intent. The design challenge is knowing when to ask versus when to just show results: a clarifying question is valuable when the answer dramatically reduces ambiguity and when the user is likely to find the question helpful rather than intrusive. Questions should be simple to answer, ideally offering a small set of clear choices. The risk of over-clarifying — asking too many questions before showing anything — is abandonment; users expect to see results quickly and will leave if they feel interrogated.
